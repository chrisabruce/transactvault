{# Shared transactions search/filter toolbar. Included from the dashboard (defaulted filter state) and the transactions list (live filter state). Posts to /app/transactions either way. Search is live: the input binds the `q` signal (`data-bind:q` — colon key, per RC.6 grammar) and each pause in typing Datastar-@gets the results fragment, which arrives as text/html and idiomorph-patches `
` in place. The input itself sits outside that div and is never re-rendered, so focus and cursor stay put. (The previous implementation submitted the whole form on a timer — every keystroke-pause was a full navigation, which froze typing and threw the cursor out of the box.) Enter still full-submits via the hidden button; select and checkbox changes full-submit too, since they move the stat-card highlight + URL, which live outside the patched region. #}
